# 02 — IR v2

The v1 IR (`packages/runner/src/reactive-interpreter/rog.ts` on the #4298
branch) got the skeleton right: a flat, topologically-sortable op list over a
closed vocabulary, `ValueRef`s into a value space, schemas on every op, and
opaque leaves by content-addressed reference. v2 keeps that skeleton and
fixes the parts that were artifacts of decompiling from the legacy graph.

## 1. What carries over unchanged

- `ValueRef` kinds `argument | opOut | const | internal` (with `internal`
  re-scoped, §2.3).
- `Op { id, kind, inputs, outSchema, detail }`, with the v1 discipline that
  **every ref an op reads is surfaced through one producer-edge view**
  (`inputsOf`) so partition/topo/CFC need no per-kind special cases.
- The `interpolate` op and the closed `expr` operator set (v1 `rog.ts`
  `EXPR_BIN_OPS`/`EXPR_UN_OPS`), including the eager-binary / prefix-unary
  disambiguation (`u-`, `u!`, …) and the `typeof` exclusion.
- The stance that ops are **pure by construction** except `effect`
  boundaries; per-op error isolation semantics (a throwing op yields
  `undefined` downstream, matching legacy per-node containment).

## 2. What changes

### 2.1 Collections carry their element graph inline

v1: `Collection { elementRog: ImplRef }` — a reference the runtime re-resolves
and re-extracts per coordinator instantiation, with a callback-injected
`resolveInner` for partition recursion.

v2: the element graph is an **inline child `Rog`** with explicit element
parameters (`element`, `index`, plus named captures as declared inputs).
Compile once; the partition recurses structurally with no callbacks; the
collection interpreter never re-parses pattern bytes.

### 2.2 Control branches are tagged, not positional

v1 encoded `when`/`unless`'s asymmetric semantics ("else returns the
condition") as a positional-array convention discovered empirically from the
builtins — a silent-wrong-branch hazard the corpus gap-map actually caught
once (the when/unless `{condition,value}` divergence).

v2: `Control { op, pred, then: ValueRef, else: ValueRef | "pred" }` — the
else-returns-predicate case is explicit in the IR. `?:`/`&&`/`||` from
pattern bodies lower to control ops with operand-return semantics pinned
(E-1 lineage).

### 2.3 Frames are first-class; no stringified causes

v1 named internals by `partialCauseToInternalName` — `JSON.stringify`ed cause
keys in a flat namespace, with per-Rog maps papering over collision risk, and
the `__patternResult` result-self-reference shape simply unmodeled (the main
lunch-poll pattern fell back on it).

v2: `internal` refs are `(frameId, name)`; nested pattern frames and
collection element frames are declared IR structure. **Result
self-references are modeled**: an egress root may be named as an input
(`ValueRef { kind: "result", path }`), which removes the v1
`unrecognized_alias` fallback class for patterns whose result tree feeds
their own computation.

### 2.4 Effects carry their full contract

v1 `Effect { sink, link? }` had no inputs at first (07 §8 F1 — the partition
had nothing to cut on) and never carried write-back targets (F4 — the
`S → handler → S` cycle was invisible to a data-flow-only partition).

v2: `Effect { sink: "render" | "pull" | "handler" | "io", inputs: ValueRef[],
writeTargets: ValueRef[], streamLink?, builtin? }`. Both edge classes exist
**by construction**: partition cut edges and the CFC labeled read-through
invariant (v1 07 §4.5) fall out of the IR instead of being extraction
preconditions.

### 2.5 Leaves carry compiler-emitted capability annotations

v1 rediscovered leaf properties at runtime by regex over `fn.toString()`.

v2: `Leaf { impl: ImplRef, caps: { async?, instantiatesPattern?,
writesInput?, needsCellContext? } }`, emitted by the compiler which has the
AST. **NG-2 discipline**: annotations are fail-closed hints. A `caps` bit can
only make an op a boundary or force cell-context wrapping — it never grants
anything. A *missing* hazard bit on a genuinely hazardous leaf degrades
exactly as v1's static-scan misses did: loudly ("Cannot store Promise"), not
silently — and the compiler, unlike a regex over source text, sees the real
AST, so the miss rate is strictly lower.

### 2.6 `fn` / `call` — the function abstraction

New in v2 (required by D-V2-LOWERING-SUBSET Tier 2):

```text
FnDef  := { id: FnId, params: string[], body: Rog }        // pure by construction
CallDetail :=
  | { kind: "call"; callee: FnId; args: ValueRef[] }        // same-bundle helper
  | { kind: "call"; builtin: StdlibId; args: ValueRef[] }   // curated stdlib method
```

- A same-bundle helper lowered by the compiler becomes one `FnDef`, invoked
  from N sites via `call` — no per-site inlining blow-up.
- The stdlib variant is a closed, versioned registry
  ([03-compiler-emission.md](./03-compiler-emission.md) §4): `StdlibId`s like
  `string.slice`, `array.join`, `math.floor`, each with oracle-verified
  JS-exact semantics. Not in the registry → the compiler emits an opaque
  leaf; the evaluator refuses unknown ids (fail-closed on both sides of the
  artifact boundary).
- Recursion (direct or mutual) → opaque leaf at compile time. `FnDef` bodies
  are Rogs, so evaluation is structural recursion with a depth guard, not a
  new machine.

### 2.7 Schemas are interned

v1 carried full `JSONSchema` inline on every op. v2 artifacts carry a schema
table; ops carry handles. (v1 anticipated this — "a later pass may intern to
a hash handle" — it just never happened, and schema identity churn was the
root of the 8.4× traverse regression class, so interning also feeds the
runtime's existing hash caches.)

### 2.8 Egress is explicit

The artifact records the egress roots — the result tree, render reads,
effect-sink inputs, handler write targets (v1 02 §4 / MA-8) — so
materialization never re-derives "what is observable" from a runtime walk.

## 3. Serialization

Content-addressed, versioned (`rogVersion`), deterministic encoding. The
artifact = `{ rogVersion, argumentSchema, resultSchema, schemaTable, ops,
fns, egress, meta }` plus the referenced opaque-leaf modules (existing
`{identity, symbol}` scheme). The compiled bundle keeps carrying source for
recompilation; the artifact-format version is the compatibility key
(D-V2-ARTIFACT §3).

## 4. The trust stance (unchanged, restated for v2's larger surface)

The ROG is **untrusted data** (v1 03-cfc §2–3, carried wholesale):

- Structure is used for scheduling and ordering; label *values* are derived
  from what the interpreter actually reads at runtime, under structurally
  enforced read isolation.
- This argument covers function lowering with no extension: a mis-lowered
  `FnDef` (malicious or buggy artifact) computes a wrong *value* — the
  author's own data — inside correct labels. It cannot induce an unsound
  label, because labels never come from the IR.
- What function lowering *does* expand is the **correctness surface**: the
  evaluator must match JS exactly, op by op and stdlib entry by stdlib
  entry. That is the differential oracle's job, and it is why the allow-list
  registry is the only gate through which vocabulary grows
  ([01-decisions.md](./01-decisions.md) D-V2-LOWERING-SUBSET invariants).
